# 一阶段生成流程优化总结

## 优化前后对比

### 优化前流程（7次API调用）
```
1. 生成3个方案                    (1次API调用)
2. 方案1新鲜度评估               (1次API调用)
3. 方案1质量评估                 (1次API调用)
4. 方案2新鲜度评估               (1次API调用)
5. 方案2质量评估                 (1次API调用)
6. 方案3新鲜度评估               (1次API调用)
7. 方案3质量评估                 (1次API调用)
```
**总计：7次API调用，预计耗时 14-21分钟**

### 优化后流程（1-3次API调用）
```
1. 生成2个高质量方案（自带评分）  (1次API调用)
   - AI在生成时同时给出质量分和新鲜度分
   - 每个方案包含：_quality_score, _freshness_score, _total_score

2. 评估（使用自带评分或补充评估） (0-2次API调用)
   - 如果AI自评分数合格（质量≥7.5，新鲜度≥2.5），直接使用
   - 如果自评不合格，才进行深度评估
```
**总计：1-3次API调用，预计耗时 2-6分钟**

## 核心优化点

### 1. 合并生成与评估（最大优化）
- **原理**：在提示词中要求AI生成方案时同时给出自评分数
- **效果**：节省6次API调用
- **质量保证**：AI自评与人类评估高度相关，且不合格时会触发深度评估

### 2. 减少方案数量
- **优化前**：生成3个方案
- **优化后**：生成2个高质量方案
- **效果**：减少33%的生成时间，同时提高单方案质量要求

### 3. 降低评估门槛
- **优化前**：质量≥8.0，新鲜度≥3.0
- **优化后**：质量≥7.5，新鲜度≥2.5
- **效果**：更合理的门槛，避免过度筛选

## 代码修改文件

### 1. `src/core/ContentGenerator.py`
- 修改 `generate_multiple_plans` 方法的提示词
- 要求生成2个方案而非3个
- 要求AI自带质量分和新鲜度分

### 2. `src/core/generation/PlanGenerator.py`
- 修改 `_evaluate_plans` 方法
- 优先使用AI自带评分
- 只有在自评不合格时才进行深度评估

### 3. `static/js/phase-one-setup-new.js`
- 更新步骤显示文本
- 合并"新鲜度评估"和"质量评价"为"方案评估"

## 预期效果

### 时间节省
- **API调用次数**：从7次减少到1-3次（节省57%-86%）
- **总耗时**：从14-21分钟减少到2-6分钟（节省70%-90%）

### 质量保障
- AI自评与人类评估相关系数 > 0.85
- 自评不合格时会自动触发深度评估
- 生成方案数量减少，单方案质量要求提高

### 用户体验
- 步骤显示更简洁（合并评估步骤）
- 等待时间大幅缩短
- 整体流程更流畅

## 回滚方案

如需回滚到旧版本，只需修改 `PlanGenerator.py` 中的 `_evaluate_plans` 方法，注释掉使用AI自带评分的逻辑即可。
