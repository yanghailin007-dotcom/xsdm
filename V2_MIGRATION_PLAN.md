## 通用迁移规范

### 1. 模板结构

所有 V2 页面必须继承 `layouts/base-v2.html`：

```jinja2
{% extends "layouts/base-v2.html" %}

{% block title %}页面标题 - 大文娱系统{% endblock %}

{% block extra_css %}
<!-- 页面特有 CSS -->
{% endblock %}

{% block content %}
<!-- 页面主体内容 -->
{% endblock %}

{% block extra_js %}
<!-- 页面特有 JavaScript -->
{% endblock %}
```

### 2. 样式规范

- 使用 V2 设计系统的 CSS 变量
- 所有组件使用 `v2-` 前缀
- 颜色使用设计系统中的变量，不要硬编码

### 3. JavaScript 保留清单

迁移时必须完整保留以下功能：

1. **API 调用**：所有 `fetch` 或 `axios` 调用的 URL 和参数
2. **事件处理**：所有 `onclick`、`onchange` 等事件处理器
3. **状态管理**：所有全局变量和状态逻辑
4. **定时器**：所有 `setInterval`、`setTimeout`
5. **DOM 操作**：所有 `getElementById`、`querySelector` 等
6. **第三方库**：所有外部 JS 库的引入和初始化

### 4. 测试清单

每个页面迁移后必须进行以下测试：

- [ ] 页面正常加载无 404/500 错误
- [ ] 所有按钮可点击
- [ ] 所有表单可提交
- [ ] 所有数据正常加载和显示
- [ ] 所有弹窗/模态框正常打开和关闭
- [ ] 响应式布局在移动端正常显示
