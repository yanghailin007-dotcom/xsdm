# 视频镜头生成修复报告

## 问题描述

在自定义视频生成模式下，AI只生成了5个镜头描述，而不是期望的25个镜头（5个单元×5个镜头/单元）。

## 根本原因

1. **AI提示词不够明确**：
   - 没有强调"必须生成恰好N个镜头"
   - 缺少具体的示例
   - 没有最小长度要求

2. **解析逻辑过于简单**：
   - 只支持一种格式（镜头1：描述）
   - 没有备用解析方案
   - 容错能力差

## 修复方案

### 1. 优化AI提示词（[`web/api/video_generation_api.py:1057-1110`](web/api/video_generation_api.py:1057-1110)）

**改进点**：
- ✅ 明确要求："必须生成恰好 {shot_count} 个镜头"
- ✅ 添加警告标记："⚠️ 必须生成完整的 {shot_count} 个镜头"
- ✅ 提供完整示例（10个镜头的完整示例）
- ✅ 设置最小长度："至少20个汉字"
- ✅ 强调独特性和连贯性

**关键改进**：
```python

### 2. 改进解析逻辑（[`web/api/video_generation_api.py:1092-1142`](web/api/video_generation_api.py:1092-1142)）

**改进点**：
- ✅ 支持多种分隔符：`:：.,、`
- ✅ 按镜头号排序和填充

- ✅ 智能补充机制

**解析流程**：
```

## 技术细节

### AI提示词优化策略

1. **明确数量要求**：使用"必须"和"恰好"等强调词
2. **视觉标记**：使用🎯和⚠️等emoji突出关键要求
3. **完整示例**：提供10个镜头的完整示例供参考
4. **质量标准**：设置最小长度和独特性要求

### 解析逻辑优化策略

1. **多层解析**：标准格式→备用格式→兜底方案
2. **容错机制**：支持多种分隔符和格式变体
3. **智能填充**：按镜头号排序，自动填充空位
4. **质量过滤**：移除空描述和过短描述

## 后续改进建议

1. **缓存机制**：缓存成功的AI响应，避免重复调用
2. **重试机制**：如果生成数量不足，自动重试
3. **质量评估**：使用NLP评估描述质量
4. **用户反馈**：允许用户调整或重新生成单个镜头
