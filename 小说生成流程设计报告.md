# 小说生成流程设计报告

## 项目概述

本项目是一个基于AI的自动化小说生成系统，支持从创意种子到完整小说的全流程生成，包括世界观构建、角色设计、情节规划、章节生成等功能。

---

## 一、系统架构

### 1.1 核心组件

#### 1.1.1 NovelGenerator（小说生成器主类）
- **位置**: `src/core/NovelGenerator.py`
- **职责**: 轻量级控制器，协调各个专门模块的工作
- **核心组件**:
  - APIClient: API客户端，负责与AI模型通信
  - ContentGenerator: 内容生成器
  - QualityAssessor: 质量评估器
  - MaterialManager: 材料管理器
  - EventBus: 事件总线

#### 1.1.2 NovelGenerationManager（小说生成管理器）
- **位置**: `web/managers/novel_manager.py`
- **职责**: Web层的任务管理和进度跟踪
- **功能**:
  - 任务状态管理
  - 进度跟踪
  - 项目数据加载
  - 质量数据管理

## 二、生成流程

### 2.1 第一阶段：设定生成（Phase One）

#### 2.1.1 流程概述
第一阶段负责生成小说的核心设定，包括世界观、角色、情节框架等基础内容。

#### 2.1.2 主要步骤

**步骤1: 初始化（0-20%）**
- 创建NovelGenerator实例
- 加载配置文件
- 初始化各管理器
- 创建检查点（支持断点续传）

**步骤2: 创意种子分析（20-40%）**
- 解析用户输入的创意种子
- 提取核心设定（coreSetting）
- 识别目标平台（targetPlatform）
- 分析题材类型（genre）

**步骤3: 世界观生成（40-60%）**
- 生成核心世界观（core_worldview）
- 构建世界规则体系
- 设定力量系统
- 建立世界背景

**步骤4: 角色设计（60-70%）**
- 主角设计（character_design）
- 配角设计
- 角色关系网络
- 角色成长路线

**步骤5: 情节规划（70-85%）**
- 生成故事大纲（story_outline）
- 全局成长规划（global_growth_plan）
- 阶段规划（stage_plans）
- 章节索引（chapter_index）

**步骤6: 写作风格指南（85-95%）**
- 生成写作风格指南（writing_style_guide）
- 平台风格适配（番茄小说风格）
- 叙事节奏控制
- 爽点设计

**步骤7: 验证与保存（95-100%）**
- 质量评估
- 数据验证
- 保存项目信息
- 更新检查点

#### 2.1.3 输出文件结构

```
小说项目/
└── {小说名}/
    ├── {小说名}_项目信息.json          # 核心项目数据
    ├── writing_style_guide.json       # 写作风格指南
    ├── plans/                         # 阶段写作计划
    │   ├── opening_stage_writing_plan.json
    │   ├── development_stage_writing_plan.json
    │   ├── climax_stage_writing_plan.json
    │   └── ending_stage_writing_plan.json
    └── chapters/                      # 章节目录（第二阶段生成）
```

### 2.2 第二阶段：章节生成（Phase Two）

#### 2.2.1 流程概述
第二阶段基于第一阶段的设定，生成具体的章节内容。

#### 2.2.2 主要步骤

**步骤1: 加载项目数据（0-30%）**
- 读取项目信息文件
- 加载世界观和角色设定
- 加载写作计划
- 初始化材料管理器

**步骤2: 章节生成循环（30-90%）**

对每一章执行以下流程：

1. **章节大纲生成**
   - 根据阶段计划生成章节大纲
   - 确定章节标题
   - 规划章节事件

2. **内容生成**
   - 调用ContentGenerator生成章节内容
   - 应用写作风格指南
   - 控制章节字数（2000-3000字）

3. **质量评估**
   - 使用QualityAssessor评估章节质量
   - 检查情节连贯性
   - 验证角色一致性

4. **优化迭代**（可选）
   - 如果质量评分低于阈值（8.5分）
   - 进行内容优化
   - 最多优化1次

5. **保存章节**
   - 保存章节内容到文件
   - 更新章节索引
   - 更新进度信息

**步骤3: 完成与收尾（90-100%）**
- 生成章节汇总
- 更新项目统计
- 清理临时文件
- 删除检查点

#### 2.2.3 章节文件结构

```json
{
  "chapter_number": 1,
  "chapter_title": "第一章 标题",
  "content": "章节正文内容...",
  "word_count": 2500,
  "outline": {
    "章节标题": "...",
    "核心事件": "...",
    "情感基调": "..."
  },
  "assessment": {
    "整体评分": 8.8,
    "优点": [...],
    "改进建议": [...]
  },
  "generation_time": "2026-02-26T14:00:00"
}
```

### 2.3 续写模式（Resume Mode）

#### 2.3.1 功能说明
支持在已有小说基础上继续生成新章节。

#### 2.3.2 触发条件
- 小说至少已生成1章
- 存在完整的项目信息
- 包含核心设定数据

#### 2.3.3 续写流程
1. 加载现有小说数据
2. 分析已生成章节
3. 提取上下文信息
4. 从指定章节开始生成
5. 保持风格和设定一致性

## 三、API接口设计

### 3.1 小说生成相关API

#### 3.1.1 启动生成任务
```
POST /api/start-generation
```

**请求参数**:
```json
{
  "creative_seed": {
    "novelTitle": "小说标题",
    "coreSetting": "核心设定",
    "genre": "题材类型",
    "targetPlatform": "fanqie"
  },
  "total_chapters": 200,
  "generation_mode": "phase_one_only",  // 或 "full_auto"
  "start_new": false  // 是否从头开始
}
```

**响应**:
```json
{
  "success": true,
  "task_id": "uuid",
  "message": "小说生成任务已启动",
  "status": "started"
}
```

#### 3.1.2 获取任务状态
```
GET /api/task/{task_id}/status
```

**响应**:
```json
{
  "task_id": "uuid",
  "status": "generating",  // initializing/generating/completed/failed
  "progress": 65,
  "current_step": "character_design",
  "created_at": "2026-02-26T14:00:00",
  "updated_at": "2026-02-26T14:05:00"
}
```

#### 3.1.3 获取任务进度
```
GET /api/task/{task_id}/progress
```

**响应**:
```json
{
  "status": "generating",
  "progress": 65,
  "current_step": "character_design",
  "timestamp": "2026-02-26T14:05:00",
  "chapter_progress": [
    {
      "chapter_number": 1,
      "status": "completed",
      "chapter_title": "第一章",
      "word_count": 2500
    }
  ]
}
```

### 3.2 项目管理API

#### 3.2.1 获取所有项目
```
GET /api/projects
```

#### 3.2.2 获取项目详情
```
GET /api/project/{title}
```

#### 3.2.3 获取章节详情
```
GET /api/project/{title}/chapter/{chapter_num}
```

#### 3.2.4 续写小说
```
POST /api/project/{title}/resume-generation
```

**请求参数**:
```json
{
  "from_chapter": 101,
  "additional_chapters": 50
}
```

## 九、错误处理

### 9.1 常见错误类型
1. **创意种子为空**: 返回错误提示
2. **生成器初始化失败**: 记录日志，返回失败状态
3. **章节生成失败**: 记录失败信息，继续下一章
4. **质量评估失败**: 使用默认评分

### 9.2 重试机制
- 最大重试次数: 3次
- JSON解析重试: 3次
- API请求超时: 自动重试
