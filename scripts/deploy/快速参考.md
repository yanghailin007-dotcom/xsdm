# 部署工具快速参考

## 🚀 快速开始

### 1️⃣ 首次使用（必须先运行）

```bash
# 初始化日志系统
scripts/deploy/init_logs.bat
```

### 2️⃣ 选择部署方式

#### 完整部署（首次或大更新）
```bash
scripts/deploy/一键部署_with_logs.bat
```
- 创建压缩包
- 上传到服务器
- 安装/更新依赖
- 启动服务
- 记录详细日志

#### 快速重启（代码小改动）
```bash
scripts/deploy/quick_restart_with_logs.bat
```
- 同步代码（使用rsync）
- 重启服务
- 记录日志
- **耗时约30秒**

#### 强制更新前端（UI未更新时）
```bash
scripts/deploy/force_update_frontend.bat
```
- 上传所有CSS/JS/HTML文件
- 清理服务器缓存
- 重启服务
- **然后强制刷新浏览器（Ctrl+F5）**

### 3️⃣ 查看日志

```bash
scripts/deploy/view_server_logs.bat
```

选择选项：
1. 应用主日志（推荐）
2. Gunicorn日志
3. 访问日志
4. 错误日志
5. 所有日志
6. 实时监控应用日志
7. 实时监控Gunicorn日志
8. 列出所有日志文件

## 📁 文件位置

### 本地日志
`scripts/deploy/deploy_logs/`
- `deploy_*.log` - 完整部署日志
- `restart_*.log` - 快速重启日志

### 服务器日志
`/home/novelapp/novel-system/logs/`
- `application.log` - 应用主日志
- `gunicorn.log` - Gunicorn日志
- `access.log` - 访问日志
- `error.log` - 错误日志

## 🔧 常用命令

### 连接服务器
```bash
ssh -i d:\work6.05\xsdm.pem root@8.163.37.124
```

### 查看实时日志
```bash
# 应用日志
tail -f /home/novelapp/novel-system/logs/application.log

# Gunicorn日志
tail -f /home/novelapp/novel-system/logs/gunicorn.log
```

### 手动重启服务
```bash
cd /home/novelapp/novel-system
source venv/bin/activate
lsof -ti:5000 | xargs kill -9
nohup gunicorn -w 2 -b 0.0.0.0:5000 --timeout 600 \
  --access-logfile logs/access.log \
  --error-logfile logs/error.log \
  web.wsgi:app > logs/gunicorn.log 2>&1 &
```

### 检查服务状态
```bash
lsof -ti:5000 && echo "服务运行中" || echo "服务未运行"
```

## ⚠️ 常见问题

### Q: 前端UI没有更新？
A: 运行 `scripts/deploy/force_update_frontend.bat`，然后强制刷新浏览器（Ctrl+F5）

### Q: 日志文件不存在？
A: 运行 `scripts/deploy/init_logs.bat` 初始化日志系统

### Q: 服务无法访问？
A: 运行 `scripts/deploy/view_server_logs.bat` 选择"4. 错误日志"查看问题

### Q: tee命令错误？
A: 已修复，请使用更新后的脚本
