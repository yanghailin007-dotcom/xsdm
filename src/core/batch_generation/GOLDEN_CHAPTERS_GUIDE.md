# 黄金三章整体生成与评估指南

## 为什么黄金三章必须整体处理？

黄金三章（第1-3章）是小说的"门面"，决定了读者是否愿意继续阅读。必须整体处理的原因：

### 1. 情绪弧线必须完整

**错误做法（逐章生成）**：
- 第1章：主角被羞辱（压抑）
- 第2章：主角获得系统（兴奋）→ 但独立生成，可能情绪不连贯
- 第3章：主角反击（爽快）→ 但可能和前两章脱节

**正确做法（整体生成）**：
```
第1章：压抑 → 突然获得 → 质疑（钩子）
第2章：尝试 → 小成功 → 期待（钩子）
第3章：冲突 → 爆发 → 满足（钩子）
```

### 2. 伏笔与呼应必须对应

- 第1章埋下的伏笔，必须在第3章得到回应
- 逐章生成可能导致"伏笔丢失"或"回应错位"

### 3. 角色状态必须自然过渡

```
普通人 → 发现金手指 → 初步掌握 → 第一次成功
（第1章）    （第1章末）    （第2章）    （第3章）
```

## 黄金三章专用评估体系

### 与普通章节评估的区别

| 对比项 | 普通章节评估 | 黄金三章评估 |
|-------|------------|------------|
| **评估重点** | 连贯性、一致性 | 吸引力、卖点契合 |
| **评估方式** | 单章独立评估 | 三章整体评估 |
| **核心指标** | 角色状态、时间线 | 钩子强度、读者留存 |
| **类型考量** | 较少 | 核心（神豪/修仙/赘婿等） |
| **读者视角** | 辅助 | 核心（模拟目标读者反应） |

### 五大评估维度

#### 1. 开篇钩子强度（20%）

**评估问题**：
- 前300字是否能15秒内抓住读者？
- 开篇是否有强烈冲突、悬念或情绪张力？
- 读者是否会产生"我想知道接下来发生什么"的想法？

**神豪文示例**：
```
✅ 优秀开篇：
"签了吧，对你我都好。"岳母将离婚协议推到陈默面前。
陈默，入赘三年，活得不如一条狗。今天，是他'刑满释放'的日子。
他拿起笔，在所有人轻蔑的目光中签下名字。
然后，他掏出手机："告诉老头子，我回来了。"

❌ 差开篇：
陈默是一个普通的年轻人，今年25岁，在一家小公司上班...
（太普通，没有冲突，读者直接划走）
```

#### 2. 类型卖点契合度（20%）

**各类型核心卖点检查清单**：

**神豪文**：
- [ ] 金钱装逼场景
- [ ] 身份反差（穷→富）
- [ ] 打脸反转
- [ ] 周围人震惊反应
- [ ] 升级爽感（花钱返利等）

**修仙文**：
- [ ] 金手指获得（神秘物品/系统）
- [ ] 初次修炼效果
- [ ] 他人轻视→震惊的反差
- [ ] 修仙世界观吸引力
- [ ] 踏上仙途的期待感

**赘婿文**：
- [ ] 屈辱铺垫到位
- [ ] 隐忍有张力
- [ ] 反转够爽
- [ ] 家人态度转变
- [ ] 身份揭露戏剧化

#### 3. 目标读者吸引力（20%）

**评估逻辑**：
1. 分析目标读者画像（年龄、性别、偏好）
2. 模拟该读者阅读黄金三章的反应
3. 评估是否满足该读者的核心期待

**示例**：
```
目标读者：20-30岁男性，喜欢逆袭爽文

模拟反应：
- 第1章："这主角也太惨了吧，跟我之前好像..."
- 第2章："终于有金手指了！我要看打脸！"
- 第3章："爽！但这就完了？我要看后续！"

评估结论：吸引力强，目标读者会追读
```

#### 4. 三章流畅度（20%）

**评估要点**：
- 三章是否像一个完整的开篇，而非三篇独立文章？
- 情绪曲线是否自然递进？
- 伏笔和呼应是否到位？

**情绪曲线标准**：

| 类型 | 第1章 | 第2章 | 第3章 |
|-----|-------|-------|-------|
| 神豪文 | 压抑→好奇 | 好奇→兴奋 | 兴奋→满足 |
| 修仙文 | 平凡→奇遇 | 质疑→小成 | 小成→震撼 |
| 赘婿文 | 屈辱→隐忍 | 隐忍→契机 | 契机→爆发 |

#### 5. 爽点/回报质量（20%）

**评估问题**：
- 第3章是否给出了足够的爽感/满足感？
- 铺垫和爆发的比例是否合适？（黄金比例：30%铺垫，70%爆发）
- 读者读完是否会期待第4章？

**神豪文示例**：
```
✅ 优秀爽点设计：
第1章：被岳母羞辱，被迫离婚（铺垫）
第2章：激活神豪系统，获得第一笔钱（转折）
第3章：前妻婚礼现场，开豪车送"祝福"，全场震惊（爆发）

❌ 差爽点设计：
第1章：被羞辱（铺垫）
第2章：获得系统，但还没用（继续铺垫）
第3章：开始花钱，但只是买了杯咖啡（爆发不足）
```

## 评估结果解读

### 评分标准

| 分数 | 等级 | 说明 | 建议 |
|-----|------|------|------|
| 9-10 | 优秀 | 开篇强力，类型纯正，读者必追 | 直接发布 |
| 7-8.9 | 良好 | 整体合格，有minor issues | 小修改后发布 |
| 5-6.9 | 合格 | 基本可读，但有明显问题 | 按建议修改 |
| <5 | 需修改 | 吸引力不足 | 建议重写 |

### 各维度评分解读

**开篇钩子 < 6分**：
- 问题：前300字抓不住读者
- 改进：直接从冲突开始，删除背景介绍

**类型契合 < 6分**：
- 问题：不像该类型的文
- 改进：增加类型必备元素（神豪就多写钱，修仙就多写修炼）

**读者吸引 < 6分**：
- 问题：目标读者无感
- 改进：研究目标读者偏好，增加共鸣点

**三章流畅 < 6分**：
- 问题：三章像三篇独立文章
- 改进：增加伏笔呼应，统一情绪弧线

**爽点质量 < 6分**：
- 问题：第3章不够爽
- 改进：加大爆发力度，增加周围人震惊反应

## 使用示例

### 完整流程

```python
from src.core.batch_generation import (
    MediumEventBatchProcessor,
    assess_golden_chapters,
    GoldenChaptersAssessor
)

# 1. 生成黄金三章
processor = MediumEventBatchProcessor(api_client, novel_generator)

result = processor.process_medium_event(
    medium_event={
        "name": "开篇黄金三章",
        "main_goal": "吸引读者，建立期待感"
    },
    chapter_range=(1, 3),  # 触发黄金三章模式
    novel_data=novel_data,
    skip_assessment=False  # 必须评估
)

if result.success:
    chapters = result.chapters
    assessment = result.assessment
    
    # 2. 查看评估结果
    print(f"黄金三章评估报告")
    print(f"==================")
    print(f"总体评分：{assessment.overall_score}/10")
    print(f"")
    print(f"各维度评分：")
    print(f"  开篇钩子：{assessment.opening_hook}/10")
    print(f"  类型契合：{assessment.type_match}/10")
    print(f"  读者吸引：{assessment.reader_attraction}/10")
    print(f"  三章流畅：{assessment.chapter_flow}/10")
    print(f"  爽点质量：{assessment.payoff_quality}/10")
    print(f"")
    
    # 3. 查看类型特定评估
    if assessment.genre_specific:
        print(f"类型评估（{assessment.genre_specific.get('type_matched', '未知')}）：")
        print(f"  情绪曲线：{assessment.genre_specific.get('emotion_curve', '')}")
        if assessment.genre_specific.get('missing_elements'):
            print(f"  缺少元素：{assessment.genre_specific['missing_elements']}")
    print(f"")
    
    # 4. 查看读者反馈模拟
    print(f"读者反馈模拟：")
    for ch, reaction in assessment.reader_reactions.items():
        print(f"  {ch}：{reaction}")
    print(f"")
    
    # 5. 查看改进建议
    if assessment.improvement_suggestions:
        print(f"改进建议：")
        for i, suggestion in enumerate(assessment.improvement_suggestions, 1):
            print(f"  {i}. {suggestion}")
    print(f"")
    
    # 6. 判断是否可发布
    if assessment.overall_score >= 7.0:
        print(f"✅ 评估通过，建议发布")
    else:
        print(f"⚠️ 评分偏低，建议修改后重新评估")
        
    # 7. 生成详细改进报告（可选）
    if assessment.overall_score < 8.0:
        assessor = GoldenChaptersAssessor(api_client)
        report = assessor.generate_improvement_guide(assessment, chapters)
        
        with open("golden_chapters_report.md", "w", encoding="utf-8") as f:
            f.write(report)
        print(f"\n详细报告已保存到 golden_chapters_report.md")
```

## 常见问题和解决方案

### Q: 黄金三章评分一直不高怎么办？

A: 检查以下常见问题：
1. **开篇太慢**：删除背景介绍，直接从冲突开始
2. **类型不纯**：增加类型必备元素（神豪就写花钱，修仙就写升级）
3. **爽点不足**：加大第3章的爆发力度，增加旁观者反应
4. **情绪断裂**：确保三章情绪递进，不要跳来跳去

### Q: 不同类型黄金三章的重点是什么？

A:
- **神豪文**：重点是"钱"和"装逼"，系统激活要快，第一次花钱要爽
- **修仙文**：重点是"升级"和"探索"，金手指要神秘，第一次突破要震撼
- **赘婿文**：重点是"反转"和"打脸"，屈辱要足，反转要狠
- **无敌文**：重点是"碾压"和"震惊"，敌人要嚣张，主角出手要轻松

### Q: 评估结果显示"缺少类型元素"怎么办？

A: 根据类型补充：
- 神豪文缺"周围人震惊"→ 增加路人/前妻的震惊反应描写
- 修仙文缺"修炼过程"→ 增加主角修炼时的感受和效果
- 赘婿文缺"身份反转"→ 增加主角身份揭露的场景
