# 项目可视化重构说明

## 📋 重构概述

本次重构将原来的"世界观查看器"分离为两个独立的功能模块：

1. **项目可视化** (`/project-viewer/<项目标题>`) - 完整的可视化功能
2. **世界观查看器** (`/worldview-viewer/<项目标题>`) - 简化的查看功能

## 🎯 重构目的

- **明确功能定位**：项目可视化侧重于展示项目的完整信息，包括角色网络、势力关系、故事事件等
- **简化世界观查看**：世界观查看器专注于展示世界观的设定信息，界面更加简洁
- **提高用户体验**：用户可以根据需求选择使用不同的查看器

## 📁 文件变更

### 新增文件

1. **web/templates/project-viewer.html**
   - 项目可视化页面模板
   - 包含完整的可视化功能：整体概览、角色网络、势力关系、故事事件、世界地图、系统设定
   - 支持编辑模式

2. **web/static/js/project-viewer.js**
   - 项目可视化页面的JavaScript逻辑
   - 实现了所有的可视化功能（从原worldview-viewer.js迁移）

3. **web/templates/worldview-viewer.html**
   - 世界观查看器页面模板（简化版）
   - 只显示世界观的设定信息：势力、地点、修炼体系、法术系统、社会制度

4. **web/static/js/worldview-viewer-simple.js**
   - 世界观查看器的JavaScript逻辑
   - 简化的数据加载和显示功能

### 修改文件

1. **web/routes/auth_routes.py**
   - 添加了 `/project-viewer/<project_title>` 路由
   - 保留了 `/worldview-viewer/<project_title>` 路由（现在指向简化版）

2. **web/templates/project-management.html**
   - 在项目卡片中添加了两个按钮：
     - 📊 项目可视化 - 跳转到完整的项目可视化页面
     - 🌍 世界观 - 跳转到简化的世界观查看器页面
   - 添加了 `viewWorldviewViewer()` JavaScript函数

### 保留文件

以下文件保持不变，仍然可以使用：

- **web/static/js/worldview-viewer.js** - 原来的完整功能（已被project-viewer.js替代）
- **web/static/css/worldview-viewer.css** - 共享的样式文件
- **web/api/worldview_api.py** - API接口保持不变

## 🔗 路由说明

### 项目可视化
```
GET /project-viewer/<项目标题>
```
- 功能：完整的项目可视化
- 视图：
  - 整体概览
  - 角色网络图
  - 势力关系图
  - 故事事件时间线
  - 世界地图
  - 系统设定
- 支持编辑模式

### 世界观查看器
```
GET /worldview-viewer/<项目标题>
```
- 功能：简化的世界观查看
- 内容：
  - 势力信息
  - 重要地点
  - 修炼体系
  - 法术系统
  - 社会制度
- 只读模式，界面简洁

## 📊 API接口

两个页面都使用相同的API接口：

```
GET /api/worldview/<项目标题>
```
返回项目的世界观数据。

```
GET /api/project/<项目标题>/with-phase-info
```
返回项目的详细信息。

```
GET /api/phase-one/products/<项目标题>
```
返回第一阶段的产物数据。

## 🎨 界面差异

### 项目可视化
- 复杂的可视化界面
- 多个视图切换
- Canvas绘制的网络图
- 编辑功能
- 适合需要深入了解项目的用户

### 世界观查看器
- 简洁的列表式界面
- 直接显示所有信息
- 只读模式
- 适合快速查看世界观设定的用户

## 🚀 使用方法

### 从项目管理页面进入

1. 访问 `/project-management`
2. 找到已完成第一阶段的项目
3. 点击"📊 项目可视化"按钮查看完整可视化
4. 点击"🌍 世界观"按钮查看简化的世界观信息

### 直接访问

```
# 项目可视化
http://localhost:5000/project-viewer/<项目标题>

# 世界观查看器
http://localhost:5000/worldview-viewer/<项目标题>
```

## 📝 注意事项

1. 两个页面都使用相同的CSS文件 (`worldview-viewer.css`)
2. 数据加载逻辑相同，都从第一阶段产物中获取数据
3. 项目可视化保留了编辑功能，可以修改世界观数据
4. 世界观查看器是只读的，专注于信息展示
5. 两个页面的JavaScript文件是独立的，互不影响
