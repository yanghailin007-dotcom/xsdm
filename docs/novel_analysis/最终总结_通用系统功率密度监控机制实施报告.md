## 📊 项目概述

**项目名称：** 《凡人：我能掠夺词条》卖点密度优化项目

**实施时间：** 2025-12-27

**核心目标：** 解决开篇20章掠夺密度严重不足的问题（仅3次掠夺，密度1.5/10章）

**解决方案：** 实施通用系统功率密度监控机制（三层防护体系）

---

## 🎯 核心成果

### 1. 问题诊断（完成✅）

**问题发现：**
- 前20章仅掠夺3次（第2、3、20章）
- 最大无掠夺间隔：16章
- 密度得分：42/100（严重不足）
- 读者期待感在第5章后断裂

**根本原因：**
- 生成内容同质化严重（第4-9章连续6章"手撕兽"）
- 规划与执行严重脱节（规划目标未达成60%）
- 缺少"卖点密度"的硬性约束

---

### 2. 解决方案（完成✅）

**三层防护体系：**

#### Layer 1: 规划层预防（设计方案完成✅）
- 在`major_event`中添加`core_selling_points_contribution`字段
- 在分解`medium_event`时自动应用密度约束
- 通用化设计，适用于所有系统流小说

#### Layer 2: 生成层控制（已实施✅）
- 创建`SystemPowerDensityMonitor`通用框架
- 支持掠夺、签到、抽卡、升级、收集5种系统类型
- 实施生成前检查、生成后验证、密度报告三大功能

#### Layer 3: 验证层保障（设计方案完成✅）
- 批量密度检查工具（待实施）
- 自动生成重写建议（待实施）

## 📝 使用指南

### 快速开始

```python
# 1. 创建监控器
from managers.SystemPowerDensityMonitor import LootSystemMonitor

monitor = LootSystemMonitor()

# 2. 生成前检查
constraints = monitor.pre_generation_check(chapter_num=21)
for constraint in constraints:
    if constraint.urgency == "critical":
        print(f"必须满足：{constraint.message}")

# 3. 生成章节后验证
result = monitor.post_generation_validate(
    chapter_num=21,
    triggered=True,
    target_name="罗刹鬼王",
    reward_name="万魂统御",
    reward_quality="紫",
    reward_type="技能类"
)

if not result["passed"]:
    print("需要重写！")
    for suggestion in result["recommendations"]:
        print(f"建议：{suggestion}")

# 4. 生成密度报告
report = monitor.generate_density_report(1, 20)
print(f"密度得分：{report['density_score']}/100")
```
