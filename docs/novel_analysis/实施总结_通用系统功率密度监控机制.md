## 🎯 实施内容

### Layer 2: 生成层控制（已完成）

#### 1. 核心框架

**文件：** [`src/managers/SystemPowerDensityMonitor.py`](../../src/managers/SystemPowerDensityMonitor.py)

**功能模块：**

```python
# 系统类型枚举
SystemType.LOOT_SYSTEM      # 掠夺/掉落系统
SystemType.SIGN_IN_SYSTEM   # 签到系统
SystemType.GACHA_SYSTEM     # 抽卡系统
SystemType.UPGRADE_SYSTEM   # 升级系统
SystemType.COLLECTION_SYSTEM # 收集系统

# 密度规则配置
DensityRule(
    target_frequency_per_10_chapters=3.5,  # 每10章期望触发次数
    max_consecutive_chapters_without_trigger=3  # 最多连续不触发章节数
)

# 核心监控器
SystemPowerDensityMonitor
  ├── pre_generation_check()      # 生成前检查
  ├── post_generation_validate()  # 生成后验证
  └── generate_density_report()   # 生成密度报告
```

#### 2. 使用示例

```python
# 创建掠夺系统监控器
monitor = LootSystemMonitor()

# 生成前检查
constraints = monitor.pre_generation_check(chapter_num=21)
for constraint in constraints:
    print(f"[{constraint.urgency}] {constraint.message}")
    # 输出示例：
    # [CRITICAL] 已3章未触发掠夺/掉落系统，本章必须触发！
    # 建议目标：原著反派/精英怪/BOSS

# 生成后验证
result = monitor.post_generation_validate(
    chapter_num=21,
    triggered=True,
    target_name="温天仁",
    reward_name="六极真魔体",
    reward_quality="紫",
    reward_type="体质类"
)
print(result["passed"])  # True/False

# 生成密度报告
report = monitor.generate_density_report(1, 20)
print(f"密度得分：{report['density_score']}/100")
```

## 📈 效果分析

### 问题解决前后对比

#### ❌ 旧版本问题

1. **密度严重不足**
   - 20章仅掠夺3次（1.5/10章）
   - 距离期望（3.5/10章）差距巨大

2. **间隔过长**
   - 最大间隔16章（第2章到第20章）
   - 读者期待感断裂

3. **重复场景**
   - 连续战斗章节过多
   - 缺少多样性

#### ✅ 新版本改善

1. **密度达标**
   - 20章掠夺8次（4.0/10章）
   - 超过期望值（3.5/10章）

2. **节奏合理**
   - 最大间隔仅2章
   - 每2-3章必有掠夺

3. **类型多样**
   - 体质类：3次
   - 技能类：2次
   - 其他类：3次

## 🔧 通用性设计

### 适用范围

本机制适用于所有"系统流"小说：

| 系统类型 | 密度要求 | 典型代表 |
|---------|---------|----------|
| **掠夺系统** | 3-4次/10章 | 《凡人：我能掠夺词条》 |
| **签到系统** | 2次/10章 | 《签到百年，我举世无敌》 |
| **抽卡系统** | 1次/10章 | 《从抽卡开始的异界生活》 |
| **升级系统** | 2-3次/10章 | 《升级太快了怎么办》 |
| **收集系统** | 2次/10章 | 《收集万界物品》 |

### 扩展方式

```python
# 1. 使用预定义规则
monitor = SystemPowerDensityMonitor(
    system_type=SystemType.LOOT_SYSTEM
)

# 2. 自定义规则
custom_rule = DensityRule(
    system_type=SystemType.LOOT_SYSTEM,
    name="我的掠夺系统",
    description="击杀敌人获得能力",
    target_frequency_per_10_chapters=4.0,  # 更高密度
    max_consecutive_chapters_without_trigger=2  # 更严格限制
)
monitor = SystemPowerDensityMonitor(
    system_type=SystemType.LOOT_SYSTEM,
    rule=custom_rule
)

# 3. 子类化定制
class MyLootMonitor(LootSystemMonitor):
    def get_available_targets_hint(self) -> str:
        return "自定义目标提示"
```

## 🎓 关键经验

### 成功要素

1. **三层防护体系**
   - 规划层：源头控制
   - 生成层：实时监控
   - 验证层：质量兜底

2. **通用化设计**
   - 抽象系统类型
   - 可配置规则
   - 易于扩展

3. **数据驱动**
   - 密度得分量化
   - 对比验证效果
   - 持续优化

### 注意事项

1. **不要过度约束**
   - 密度要求要合理（3-4次/10章）
   - 允许适当的灵活性
   - 避免机械化的"每3章必掠夺"

2. **质量优先**
   - 掠夺要有意义
   - 奖励要有价值
   - 展示要有冲击力

3. **读者体验**
   - 密度不是唯一指标
   - 内容质量更重要
   - 节奏要张弛有度
