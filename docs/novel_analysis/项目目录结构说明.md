# 项目目录结构说明

## 标准项目目录结构

所有小说项目都遵循统一的目录结构设计，确保产物文件存储位置清晰可预测。

```
小说项目/{小说名称}/
├── chapters/                    # 章节内容
├── characters/                 # 角色设计文件
├── worldview/                  # 世界观设定文件
├── planning/                   # 写作计划文件
│   ├── {标题}_写作计划.json
│   └── {标题}_故事线.json
├── stage_writing_plans/         # 阶段写作计划
│   ├── opening_stage_writing_plan.json
│   ├── development_stage_writing_plan.json
│   ├── climax_stage_writing_plan.json
│   └── ending_stage_writing_plan.json
├── market_analysis/            # 市场分析文件
│   └── 市场分析.json
├── materials/                  # 原料目录（可选）
│   ├── market_analysis/
│   └── worldview/
├── project_info/               # 项目信息文件
├── quality_reports/            # 质量报告
├── events/                     # 事件记录
├── stage_plan/                  # 阶段规划
└── mindset/                    # 角色心境文件
```

## 产物加载优先级

产物加载遵循以下优先级，确保能找到各种历史版本的产物文件：

### 1. 从 manager 的 quality_data 加载（最高优先级）
- **路径**: `manager.get_novel_detail(title).quality_data.writing_plans`
- **说明**: 这是在服务器端已经加载到内存的数据，获取速度最快
- **用途**: 提供最新的写作计划数据

### 2. 从标准项目结构加载（推荐）
- **世界观**: `worldview/` 或 `materials/worldview/`
- **角色设计**: `characters/`
- **写作计划**: `planning/{标题}_写作计划.json` 或 `stage_writing_plans/`
- **市场分析**: `market_analysis/` 或 `materials/market_analysis/`
- **故事线**: 从写作计划中提取或 `planning/{标题}_故事线.json`

### 3. 从旧目录结构加载（兼容性）
- **路径**: `小说项目/{标题}_第一阶段设定/产物/`
- **说明**: 支持旧版本的项目结构

### 4. 从第一阶段设定文件加载（最后的兼容方案）
- **路径**: `小说项目/{标题}_第一阶段设定/{标题}_第一阶段设定.json`
- **说明**: 最旧的项目结构

## ProductLoader 类

新的 `ProductLoader` 类统一了所有产物加载逻辑：

```python
loader = ProductLoader(title, logger)
products = loader.load_all_products()
```

**加载方法**:
- `load_all_products()`: 统一的入口，按优先级加载所有产物
- `_load_from_quality_data()`: 从manager内存加载
- `_load_from_standard_structure()`: 从标准目录结构加载
- `_load_from_legacy_structure()`: 从旧目录结构加载
- `_load_from_phase_one_file()`: 从第一阶段设定文件加载

## 产物文件命名规范

### 标准命名（推荐）
- 世界观: `{标题}_世界观.json`
- 角色设计: `{标题}_角色设计.json`
- 写作计划: `{标题}_写作计划.json`
- 故事线: `{标题}_故事线.json`
- 市场分析: `市场分析.json`

### 阶段写作计划命名
- `opening_stage_writing_plan.json`
- `development_stage_writing_plan.json`
- `climax_stage_writing_plan.json`
- `ending_stage_writing_plan.json`

## 产物完整性验证

必需的产物类别：
1. **worldview** (世界观设定) - 最重要的基础
2. **characters** (角色设计) - 角色信息
3. **writing** (写作计划) - 包含故事线信息
4. **storyline** (故事线) - 可以从写作计划提取
5. **growth** (成长路线) - 可以从写作计划提取
6. **market** (市场分析) - 可选

## 目录清理建议

如果需要清理混乱的旧结构，可以：
1. **保留标准结构**: 上述目录结构
2. **删除旧目录**: `{标题}_第一阶段设定/` 及其子目录
3. **更新索引文件**: 确保所有产物引用指向标准目录
