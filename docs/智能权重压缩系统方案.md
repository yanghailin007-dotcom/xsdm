# 智能权重压缩系统 - 完整方案

## 📋 问题诊断

你的观察非常准确！原来的**简单距离压缩**确实不适合小说创作，主要问题：

1. **主角信息被过度压缩** - 即使是50章前的主角信息，对当前创作仍很重要
2. **主线剧情丢失** - 远距离的主线情节被简化成极简信息，导致故事连贯性丧失
3. **重要转折点被忽略** - 关键的情节转折、伏笔设置可能被当作普通事件压缩
4. **一刀切的权重** - 不区分内容的创作价值，只看章节距离

## 🎯 智能权重压缩方案

### 核心思想

**基于内容重要性而非章节距离进行压缩**

```
内容权重 = 基础权重 × 时间衰减因子 × 情节影响因子
```

### 权重配置体系

#### 1. 内容类型权重（Content Types）

| 类型 | 权重 | 说明 | 示例 |
|------|------|------|------|
| **protagonist** | 1.0 | 主角相关 | 主角的能力、心理、成长历程 |
| **main_plot** | 0.9 | 主线剧情 | 突破境界、获得传承、重大抉择 |
| **major_event** | 0.8 | 重大事件 | 生死决战、情感爆发、身世揭秘 |
| **relationship** | 0.7 | 重要关系 | 师徒、爱人、生死之交 |
| **character_arc** | 0.6 | 角色成长 | 心境变化、性格塑造 |
| **subplot** | 0.4 | 支线剧情 | 辅助故事线、过渡情节 |
| **supporting_char** | 0.3 | 配角信息 | 配角的背景和状态 |
| **background** | 0.2 | 背景设定 | 世界观、修炼体系 |
| **minor_event** | 0.1 | 次要事件 | 日常琐事、过场 |

#### 2. 时间衰减策略

**不同内容类型有不同的衰减速度**：

- **关键内容（protagonist, main_plot, major_event）**
  - 衰减率：每章2%
  - 最低保留：30%权重
  - 即使100章前的主角信息，仍保留30%权重

- **普通内容（其他类型）**
  - 衰减率：每章5%
  - 最低保留：10%权重
  - 远距离内容快速衰减到最低权重

**数学公式**：
```
时间因子 = max(最低权重, (1 - 衰减率) ^ 章节距离)
```

#### 3. 情节影响因子

| 影响类型 | 因子 | 触发条件 |
|---------|------|----------|
| **plot_twist** | 1.2× | 包含"转折"、"反转"、"意外" |
| **emotional_peak** | 1.1× | 包含"高潮"、"激动"、"情感" |
| **setup** | 0.8× | 包含"伏笔"、"铺垫"、"准备" |
| **payoff** | 1.0× | 包含"回收"、"揭晓"、"结果" |

### 压缩策略

根据**综合权重**和**目标tokens**动态选择策略：

#### 1. preserve_all（完全保留）
- 条件：有关键内容 && 空间充足（compression_ratio > 0.8）
- 行为：完整保留所有内容

#### 2. smart_reduce（智能简化）
- 条件：空间中等（0.5 < compression_ratio ≤ 0.8）
- 行为：
  - 关键元素（is_critical=true）：完全保留
  - 高权重元素（weight > 0.6）：保留70%
  - 中权重元素（0.3 < weight ≤ 0.6）：保留40%
  - 低权重元素：可能丢弃

#### 3. essence_only（精华提取）
- 条件：空间紧张（0.2 < compression_ratio ≤ 0.5）
- 行为：只保留权重最高的前5个元素，每个元素最多200字

#### 4. minimal（最小压缩）
- 条件：空间极小（compression_ratio ≤ 0.2）
- 行为：只保留关键元素的标识信息（name、title、status）

## ⚖️ 两种方案的选择

### 使用传统LayeredContextManager

- ✅ 适合：通用文档处理、技术文档、日志压缩
- ✅ 优势：简单高效、压缩率高、性能好
- ❌ 缺点：不考虑内容语义、可能丢失重要信息

### 使用IntelligentWeightedCompressionManager

- ✅ 适合：**小说创作、剧本写作、故事续写**
- ✅ 优势：**保留故事连贯性、保护关键情节、智能权重**
- ⚠️ 注意：占用更多空间、计算更复杂

### 推荐策略：**混合使用**

```python
def hybrid_compression(context, current_chapter, context_chapter):
    """混合压缩策略"""

    # 1. 先用智能压缩识别关键内容
    intelligent_manager = IntelligentWeightedCompressionManager()
    analysis = intelligent_manager._analyze_context_weights(
        context, current_chapter, context_chapter
    )

    # 2. 提取关键元素
    critical_elements = {
        key: context[key]
        for key in analysis["critical_elements"]
    }

    # 3. 对非关键元素使用传统压缩
    traditional_manager = LayeredContextManager()
    non_critical = {
        key: value
        for key, value in context.items()
        if key not in analysis["critical_elements"]
    }
    compressed_non_critical = traditional_manager.compress_context(
        non_critical, current_chapter, context_chapter
    )

    # 4. 合并结果
    return {**critical_elements, **compressed_non_critical}
```
