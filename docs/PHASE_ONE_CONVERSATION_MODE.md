# 一阶段多轮对话模式（Kimi 专用）

## 概述

针对 Kimi API 的 **256K 上下文窗口** 和 **上下文缓存机制**，系统实现了专属的多轮对话优化模式。将整个一阶段（7个主要步骤）整合为单次连续对话，预计节省 **60-70%** 的 Token 成本。

## 模型参数

| 模型 | 上下文长度 | 缓存命中价格 | 缓存未命中价格 | 输出价格 |
|------|-----------|-------------|---------------|---------|
| **kimi-k2.5** | 256K (262,144) | ¥0.70/1M | ¥4.00/1M | ¥21.00/1M |
| kimi-k2-0905-preview | 256K | ¥1.00/1M | ¥4.00/1M | ¥16.00/1M |
| kimi-k2-turbo-preview | 256K | ¥1.00/1M | ¥8.00/1M | ¥58.00/1M |

**推荐**: kimi-k2.5 - 最佳性价比，支持完整多轮对话优化

## 配置示例

使用 `config/kimi_k25_config.json`:

```json
{
  "api_endpoints": {
    "kimi": [
      {
        "name": "kimi-k2.5-primary",
        "api_url": "https://api.moonshot.cn/v1/chat/completions",
        "api_key": "your-api-key",
        "model": "kimi-k2.5",
        "priority": 1,
        "enabled": true
      }
    ]
  },
  "models": {
    "kimi": "kimi-k2.5"
  },
  "default_provider": "kimi",
  "use_conversation_mode_for_kimi": true
}
```

**注意**: kimi-k2.5 必须设置 `temperature=1.0`，不支持其他温度值。

## 启用条件

多轮对话模式仅在以下条件下自动启用：

1. **Provider 为 `kimi`** - 通过 API 客户端配置判断
2. **模块导入成功** - `PhaseOneConversation` 模块可用
3. **配置未禁用** - `use_conversation_mode_for_kimi` 不为 `false`

## 配置选项

在生成配置中可选禁用：

```json
{
  "use_conversation_mode_for_kimi": false
}
```

默认值为 `true`，即自动启用。

## 降级机制

如果多轮对话模式失败，系统会自动回退到标准分步调用模式，确保生成流程不中断。

## 注意事项

1. **上下文污染**：如果某一轮输出格式异常，可能影响后续轮次。系统会在每轮后验证 JSON 格式。
2. **长上下文衰减**：虽然 256K 足够，但超长上下文后期可能有信息遗忘。关键信息会在后续轮次适当重申。
3. **错误隔离**：单轮对话失败会导致整个会话重置。建议在网络稳定环境下使用。

- **Kimi 端点**: 启用多轮对话模式
