# 世界观可视化系统使用指南

## 🌍 功能概述

世界观可视化系统是一个全新的交互式界面，让你能够直观地查看、编辑和管理小说的世界观设定。系统提供多种可视化视图，帮助作者更好地理解和构建世界。

## ✨ 主要特性

### 1. 🗺️ 地图视图
- **交互式地图**：使用Canvas绘制的动态地图，展示世界地理布局
- **势力区域**：不同颜色的区域表示各势力的领土范围
- **地点标记**：重要的地理位置以图标形式标注
- **悬停提示**：鼠标悬停显示地点详细信息
- **一键聚焦**：点击地点或势力快速定位

### 2. 📜 时间轴视图
- **历史时间线**：展示世界历史发展脉络
- **重大事件**：标记关键历史节点
- **阶段划分**：清晰显示不同历史时期
- **事件描述**：每个事件的详细说明

### 3. 🔗 势力关系视图
- **关系网络图**：可视化展示势力之间的关系
- **连线类型**：
  - 实线：联盟关系（绿色）
  - 虚线：敌对关系（红色）
  - 点线：中立关系（灰色）
- **势力节点**：图标化显示各势力
- **互动效果**：鼠标悬停查看详细信息

### 4. 📖 规则系统视图
- **修炼体系**：等级划分、修炼方法
- **法术系统**：法术分类、使用规则
- **社会制度**：势力等级、经济体系
- **世界规则**：基本规则、特殊限制

### 5. ✏️ 编辑模式
- **可视化编辑**：直接在界面上修改世界观
- **实时预览**：修改即时反映在视图中
- **导入导出**：支持JSON格式的数据导入导出
- **版本保存**：自动保存修改历史

## 🚀 使用方法

### 访问界面

1. **从项目管理页面进入**
   - 访问 `/project-management`
   - 找到你想要查看世界观的项目
   - 点击"世界观"按钮进入可视化界面

2. **直接访问**
   ```
   http://localhost:5000/worldview-viewer/<项目名称>
   ```

### 基本操作

#### 视图切换
- 点击左侧控制面板的视图按钮切换不同视图
- 当前激活的视图会高亮显示

#### 筛选功能
- 使用筛选标签快速过滤特定势力
- 点击标签启用/禁用筛选

#### 编辑世界观
1. 点击右上角"编辑模式"按钮
2. 在弹出的编辑器中修改内容
3. 点击"保存"按钮保存修改
4. 或点击"取消"放弃修改

#### 添加新势力
1. 进入编辑模式
2. 在"势力管理"部分点击"+ 添加势力"
3. 填写势力信息：
   - 名称
   - 描述
   - 颜色（选择代表色）
   - 图标（选择表情符号）
   - 力量值（1-100）
   - 领土列表
   - 势力关系

#### 添加新地点
1. 进入编辑模式
2. 在"重要地点"部分点击"+ 添加地点"
3. 填写地点信息：
   - 名称
   - 描述
   - 所属势力
   - 地图坐标（x, y，百分比）
   - 重要性等级（1-5星）

#### 导入导出
- **导出**：点击"导出"按钮将世界观保存为JSON文件
- **导入**：点击"导入"按钮上传已有的世界观数据

## 🎨 自定义配置

### 修改默认数据

编辑 `web/static/js/worldview-viewer.js` 中的 `worldviewData` 对象来修改默认显示的世界观数据。

### 调整样式

修改 `web/static/css/worldview-viewer.css` 中的CSS变量来自定义配色方案：

```css
:root {
  --primary-color: #667eea;
  --secondary-color: #764ba2;
  --bg-dark: #0f172a;
  --bg-darker: #020617;
  /* ... 其他颜色变量 */
}
```

## 💡 使用技巧

1. **规划地图布局**
   - 先确定主要势力的地理位置
   - 使用百分比坐标精确定位地点
   - 考虑势力之间的战略关系

2. **设计势力关系**
   - 创建清晰的敌友关系
   - 设置合理的势力对比
   - 考虑剧情发展的需要

3. **构建时间线**
   - 从上古到现在的历史脉络
   - 标记关键转折点
   - 为后续剧情埋下伏笔

4. **完善规则系统**
   - 修炼等级要平衡
   - 法术系统要合理
   - 社会制度要自洽

## 🔧 技术实现

### 前端技术
- **HTML5**：页面结构
- **CSS3**：样式和动画
- **JavaScript (ES6+)**：交互逻辑
- **Canvas API**：地图和关系图绘制

### 后端技术
- **Flask**：Web框架
- **Python**：后端逻辑
- **JSON**：数据存储格式

### API接口

#### 获取世界观数据
```
GET /api/worldview/<project_title>
```

#### 保存世界观数据
```
POST /api/worldview/<project_title>
Content-Type: application/json

{
  "worldName": "...",
  "worldDescription": "...",
  ...
}
```

#### 导出世界观数据
```
GET /api/worldview/<project_title>/export
```

#### 导入世界观数据
```
POST /api/worldview/<project_title>/import
Content-Type: multipart/form-data

file: <JSON文件>
```

## 🐛 常见问题

### Q: 地图显示不正常？
A: 确保浏览器支持Canvas，并且窗口大小合适。尝试刷新页面或调整浏览器窗口大小。

### Q: 保存失败？
A: 检查是否已登录，并确保项目目录有写入权限。

### Q: 导入的数据格式不对？
A: 确保导入的JSON文件符合系统定义的数据结构。参考上面的数据结构说明。

### Q: 势力关系图显示混乱？
A: 减少势力数量，或者调整势力在画布上的布局位置。
