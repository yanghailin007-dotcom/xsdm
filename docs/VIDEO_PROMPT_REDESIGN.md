# 视频生成提示词重新设计文档

## 问题分析

### 原有问题
之前的提示词生成逻辑存在问题：
- **输入**：只包含小说的核心设定（如"非人流 + 幕后养成 + 吞噬进化"）
- **问题**：这些是高级别的剧情概念，不是具体的场景描述
- **结果**：生成的提示词过于抽象，无法直接用于视频生成

### 实际需求
对于视频生成，我们需要：
- **具体的场景描述**：环境、角色动作、情绪、视觉风格等细节
- **类似小说场景生成**：像生成小说章节时使用的详细场景提示词
- **可执行的镜头指令**：每个镜头都应有清晰的视觉描述

## 解决方案

### 新架构设计

```
小说数据
    ↓
事件提取器 (EventExtractor)
    ↓
中级事件列表 (Medium Events)
    ↓
视频场景提示词生成器 (VideoScenePrompts)
    ↓
详细的场景级提示词
    ↓
AI视频生成工具
```

### 核心组件

#### 1. VideoScenePrompts 类
**文件**: `src/prompts/VideoScenePrompts.py`

**主要功能**:
- 从中级事件生成具体的场景描述
- 包含环境、角色、视觉风格、镜头设计等详细信息
- 支持三种视频类型（长篇剧集、短片、短视频）

**关键方法**:

```python
# 为单个中级事件生成场景提示词
generate_scene_prompt(medium_event, novel_data) -> str

# 为整个视频项目生成提示词
generate_video_type_prompt(
    selected_events,
    selected_characters, 
    video_type,
    novel_data
) -> str
```

#### 2. API接口改进
**文件**: `web/api/video_generation_api.py`

**修改点**:
1. **新增参数**:
   - `selected_events`: 选中的事件列表（包含中级事件）
   - `selected_characters`: 选中的角色列表

2. **自动提取逻辑**:
   - 如果未指定事件，自动提取所有中级事件
   - 如果未指定角色，自动提取主要角色

3. **元数据返回**:
   - 预计镜头数
   - 预计总时长
   - 选中的事件和角色数量

## 提示词结构

### 新提示词包含的内容

#### 1. 项目信息
- 小说标题
- 视频类型（长篇剧集/短片/短视频）
- 目标时长
- 风格特点

#### 2. 内容概览
**选中事件**:
```
1. 锈剑苏醒，废物祭旗 (起因): 暂无描述...
2. 强制杀戮，饮血进化 (发展): 暂无描述...
```

**出场角色**:
```
- 赤 (主角): ...
- 姜清歌 (配角): ...
```

#### 3. 分集策略

**长篇剧集 (long_series)**:
- 每个中级事件独立成为一集
- 起因事件: 5个镜头，约2分钟
- 发展事件: 8个镜头，约3.5分钟
- 高潮事件: 15个镜头，约7.5分钟（时长加倍）
- 结局事件: 4个镜头，约1.8分钟

**短片 (short_film)**:
- 从所有事件中选择3-8个核心重大事件
- 每个事件5-7个镜头
- 总时长5-30分钟

**短视频 (short_video)**:
- 每个重大事件 = 1个短视频
- 5-7个快速镜头
- 前3秒必须有钩子

#### 4. 镜头设计要求

根据叙事阶段设计不同的镜头序列：

**起因阶段**:
- 开场使用全景或大远景建立环境
- 镜头缓慢推近，营造渐进的紧张感
- 时长: 4-5秒/镜头
- 节奏: 缓慢，信息渐进释放

**发展阶段**:
- 使用中景和近景展示角色互动
- 镜头运动多样，保持视觉新鲜感
- 时长: 3-4秒/镜头
- 节奏: 明快，信息密集

**高潮阶段**:
- 大量使用特写和大特写强调关键细节
- 快速剪辑配合激烈运镜（手持、推拉、环绕）
- 时长: 2-6秒/镜头（变化幅度大）
- 节奏: 极快，情绪爆发
- 特效: 高潮时刻可使用慢动作突出决定性瞬间

**结局阶段**:
- 使用全景和中景展示结果
- 镜头缓慢拉远，释放情绪
- 时长: 4-5秒/镜头
- 节奏: 舒缓，留下余韵

#### 5. 视觉风格

根据小说类型自动适配：

**玄幻**:
- 整体风格: 东方幻想风格，特效华丽
- 色彩基调: 金色、紫色、青色为主，仙气飘渺
- 光影效果: 法术光效，环境光柔和
- 构图风格: 对称构图，大场景与小细节结合

**武侠**:
- 整体风格: 古风写实，动作流畅
- 色彩基调: 自然色调，棕绿色为主
- 光影效果: 自然光，晨光夕照
- 构图风格: 动态构图，强调动作线条

**都市**:
- 整体风格: 现代写实
- 色彩基调: 城市色调，霓虹灯光
- 光影效果: 城市夜景灯光，室内照明
- 构图风格: 现代构图，多视角

**科幻**:
- 整体风格: 未来感，科技美学
- 色彩基调: 蓝白为主，金属质感
- 光影效果: 冷光，全息投影光效
- 构图风格: 几何构图，科技感强

#### 6. 音频设计

**背景音乐**:
- 起因: 渐进式紧张音乐，缓慢增强
- 发展: 节奏明快音乐，中等强度
- 高潮: 紧张激烈音乐，高强度
- 结局: 舒缓释放音乐，逐渐减弱

**音效**:
- 根据场景动作添加相应音效
- 音频节奏与镜头节奏完美匹配

## 使用流程

### 前端操作流程

1. **选择小说和视频类型**
2. **选择事件和角色**
   - 在左侧面板勾选想要生成的事件
   - 在角色标签页勾选想要出场的角色
3. **点击"确认选择并生成提示词"**
4. **查看生成的提示词**
   - 包含详细的场景描述
   - 包含镜头设计建议
   - 包含音频设计方案
5. **生成视频**
   - 使用提示词生成分镜头脚本
   - 逐个或批量生成视频


## 优势对比

### 旧方案 vs 新方案

| 维度 | 旧方案 | 新方案 |
|------|--------|--------|
| **输入** | 小说核心设定（高级别概念） | 具体的中级事件和角色 |
| **输出** | 抽象的剧情概念 | 详细的场景级描述 |
| **可执行性** | 无法直接用于视频生成 | 可直接输入AI工具 |
| **细节程度** | 缺少视觉细节 | 包含镜头、音效、视觉风格 |
| **用户体验** | 需要手动补充大量细节 | 自动生成完整指令 |

## 技术细节

### 数据流

```
1. 用户选择事件和角色
   ↓
2. 前端发送POST请求到 /api/video/generate-prompt
   ↓
3. 后端接收请求，提取小说数据
   ↓
4. 如果未指定事件，使用EventExtractor自动提取所有中级事件
   ↓
5. 使用VideoScenePrompts生成详细的场景级提示词
   ↓
6. 返回提示词和元数据（镜头数、时长等）
   ↓
7. 前端显示提示词，用户可编辑
   ↓
8. 用户确认后，调用 /api/video/generate-storyboard 生成分镜头
   ↓
9. 逐个或批量生成视频
```

### 关键函数

**_estimate_total_shots(events, video_type)**
```python
def _estimate_total_shots(events: List[Dict], video_type: str) -> int:
    """估算总镜头数"""
    if video_type == "long_series":
        shots_map = {"起因": 5, "发展": 8, "高潮": 15, "结局": 4}
        return sum(shots_map.get(e.get("stage", "发展"), 8) for e in events)
    elif video_type == "short_film":
        return len(events) * 6
    else:  # short_video
        return len(events) * 6
```

**_estimate_total_duration(events, video_type)**
```python
def _estimate_total_duration(events: List[Dict], video_type: str) -> float:
    """估算总时长（分钟）"""
    if video_type == "long_series":
        duration_map = {"起因": 2.0, "发展": 3.5, "高潮": 7.5, "结局": 1.8}
        return sum(duration_map.get(e.get("stage", "发展"), 3.5) for e in events)
    elif video_type == "short_film":
        return len(events) * 3.5
    else:  # short_video
        return len(events) * 1.0
```

## 总结

这次重新设计彻底解决了原有提示词过于抽象的问题：

✅ **从抽象到具体**: 从核心设定改为具体场景描述
✅ **从概念到可执行**: 从剧情概念改为AI可直接使用的指令
✅ **从简单到完整**: 从简单描述改为包含镜头、音效、视觉的完整方案
✅ **从手动到自动**: 从需要手动补充细节到自动生成完整提示词

新的提示词生成系统与小说场景生成系统保持一致，都是基于具体的、可视化的场景描述，确保了生成质量。
