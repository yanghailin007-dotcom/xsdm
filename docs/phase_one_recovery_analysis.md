# 一阶段恢复模式问题分析与改进方案

## 当前问题

### 1. 恢复状态不直观
- **现象**: 用户需要在"我的作品"页面下拉菜单中才能看到"续写"按钮
- **原因**: 
  - 状态显示不清晰（completed/generating/paused 三种状态）
  - 没有专门的"可续写"视觉标识
  - 首页的"继续生成"区域与"我的作品"页面分离

### 2. 无法正确恢复写作
- **现象**: 用户点击续写后无法正常恢复生成
- **原因**: 
  - 后端两个API的判断逻辑不一致
  - `/api/projects/with-phase-status` 使用 ProductLoader 检查7个产物
  - `/api/phase-one/continue-to-phase-two` 只检查 quality_data 和 core_worldview

## 改进方案

### 1. 视觉改进
- 在"我的作品"页面添加"可续写"状态标签
- 在"已暂停"状态下显示明显的"继续生成"按钮
- 在首页"继续生成"区域增加跳转链接

### 2. 逻辑修复
- 统一使用 ProductLoader 判断第一阶段完成状态
- 统一标准：必须完成7个核心产物（worldview, factions, characters, growth, writing, storyline, market）

### 3. 体验优化
- 点击"续写"后直接进入第二阶段生成界面
- 自动加载已完成的第一阶段产物
- 显示清晰的恢复进度提示

## 实施步骤

1. 修复后端API判断逻辑不一致问题
2. 改进前端状态显示
3. 优化续写操作流程
4. 添加恢复进度提示
