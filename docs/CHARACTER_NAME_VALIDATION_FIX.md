# 角色名称验证修复报告

## 问题描述

在小说生成过程中，包含英文翻译的角色名称（如"魔渊"、"姜清雪"、"赵无极"）被错误地拒绝添加到角色发展表中。

错误日志：
```
❌ 名称长度不合理: 12 字符
❌ 名称长度不合理: 19 字符
❌ 名称长度不合理: 15 字符
```

## 根本原因

原有的名称长度验证逻辑直接计算完整字符串长度，包括英文括号和拼音翻译，导致包含翻译的角色名称超过长度限制（原限制为6个字符）。

## 修复方案

### 1. 智能名称解析（第2988-3014行）

在 [`_should_accept_new_character()`](../src/managers/WorldStateManager.py:2965) 方法中实现了智能名称解析：

```python
# 提取纯中文名称部分进行长度验证
pure_chinese_name = character_name

# 如果包含英文括号，提取括号前的中文部分
if '(' in character_name:
    pure_chinese_name = character_name.split('(')[0].strip()
elif '（' in character_name:
    pure_chinese_name = character_name.split('（')[0].strip()

# 分别验证
chinese_name_length = len(pure_chinese_name)  # 2-6个字符
full_name_length = len(character_name)        # 最多20个字符
```

**支持格式**：
- `中文名` ✅
- `中文名` ✅
- `中文名` ✅

### 2. 扩展修仙小说字符集（第3037-3080行）

在双字名称验证中扩展了允许的字符集：

**新增修仙小说常见字**：
- 特殊字：`墨`, `魔`, `玄`, `虚`, `云`, `风`, `雷`, `电`, `冰`, `火`, `水`, `土`, `木`, `金`
- 自然字：`天`, `地`, `人`, `神`, `仙`, `鬼`, `妖`, `圣`, `皇`, `帝`, `王`, `君`, `主`, `尊`
- 动物字：`龙`, `凤`, `虎`, `豹`, `狼`, `鹰`, `鹤`, `蛇`, `狐`, `兔`, `马`, `牛`, `羊`
- 修仙人名用字：`渊`, `凡`, `清`, `雪`, `无`, `极`, `青`, `元`, `沛`, `灵`, `慕`, `衍`

**验证逻辑优化**：
1. 首先检查首字是否为常见姓氏或修仙特殊字
2. 如果不是，检查第二个字是否为人名用字
3. 将严格的拒绝改为警告，提高通过率
