## 🔧 故障排查

### 问题 1：页面无法加载

**症状：** 访问 http://localhost:5000 显示错误

**排查步骤：**
1. 检查服务是否运行：`python web_server.py` 仍在运行
2. 检查端口 5000 是否被占用
3. 检查防火墙是否阻止

**解决方案：**
```bash
# 改用其他端口
# 编辑 web_server.py 最后一行
app.run(host='0.0.0.0', port=8000)
```

### 问题 2：API 返回 500 错误

**症状：** 生成时出错

**排查步骤：**
1. 查看服务器日志
2. 检查 test_e2e_with_mock_data.py 是否可正常导入
3. 检查 MockAPIClient 是否正常工作

**解决方案：**
```bash
# 测试模拟数据源
python test_e2e_with_mock_data.py
```

### 问题 3：样式加载失败（白色页面）

**症状：** 页面没有样式

**排查步骤：**
1. 检查浏览器控制台错误
2. 确保 `static/css/style.css` 存在
3. 清空浏览器缓存

**解决方案：**
```bash
# 检查文件
dir d:\work6.03\static\css\
```

### 问题 4：章节内容为空

**症状：** 正文显示"加载中..."

**排查步骤：**
1. 检查是否已成功生成章节
2. 查看 API 响应是否正确
3. 检查浏览器控制台错误

**解决方案：**
```bash
# 手动测试 API
curl http://localhost:5000/api/chapters
```

## 📞 常见问题

### Q: 如何在其他电脑访问？
**A:** 使用 IP 地址代替 localhost
```
http://<服务器IP>:5000
```

### Q: 如何改变样式风格？
**A:** 编辑 `static/css/style.css`
```css
/* 改变主色调 */
--primary-color: #667eea;  /* 改为其他颜色 */
```

### Q: 如何添加新功能？
**A:** 
1. 后端：编辑 `web_server.py` 添加新 API
2. 前端：编辑对应 `.html` 和 `.js` 文件

### Q: 如何与真实 API 集成？
**A:** 替换 `MockAPIClient` 为真实 `APIClient`
```python
# 在 web_server.py 中
from APIClient import APIClient  # 替换为真实客户端
scenario.api_client = APIClient(api_key="...")
```
