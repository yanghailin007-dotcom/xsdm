### 1.2 步骤职责划分

| 步骤 | 名称 | 核心产出 | 关键动作 |
|------|------|----------|----------|
| 1 | 选集 | 选中的集数列表 | 勾选集数 |
| 2 | 角色剧照 | 角色形象图库 | 上传/生成角色图 |
| **3** | **故事节拍** | **场景叙事框架** | **AI生成+人工审核** |
| 4 | 分镜生成 | 完整分镜JSON | AI生成镜头+Veo提示词+人工审核 |
| 5 | 视频生成 | 视频片段 | 调用Veo生成 |
| 6 | 配音制作 | 音频文件 | TTS配音 |
| 7 | 导出 | 最终成片 | 合成导出 |

### 2.3 AI生成Prompt

```markdown
你是一个专业的短剧编剧。请根据以下集数内容，生成故事节拍(Story Beats)。

## 输入信息
集数标题：{{EPISODE_TITLE}}
集数内容：
{{EPISODE_CONTENT}}

角色设定：
{{CHARACTERS}}

总时长要求：{{TOTAL_DURATION}}秒

## 输出要求

1. **场景划分**
   - 将集数内容划分为8-12个场景
   - 每个场景时长4-10秒
   - 总时长严格等于{{TOTAL_DURATION}}秒

2. **每个场景包含**
   - sceneNumber: 场景序号
   - sceneTitleCn/En: 中英文标题
   - storyBeatCn/En: 叙事目的（一句话）
   - startTime/endTime: 时间轴位置
   - durationSeconds: 时长
   - emotionalArc: 情绪曲线（start→middle→end）
   - dialogues: 对白列表

3. **对白设计原则**
   - 每个场景至少1句对白
   - 对白要推动剧情或揭示人物
   - 提供中英文双语
   - 标注语气/情绪

4. **三幕结构建议**
   - 第一幕（0-30%）：建立场景、人物、冲突
   - 第二幕（30-70%）：冲突升级、内心挣扎
   - 第三幕（70-100%）：高潮、解决、悬念

只输出JSON，不要解释。
```

### 3.3 AI生成Prompt（按场景）

```markdown
你是一个AI视频分镜工程师。请根据以下场景设计，生成分镜细节。

## 场景设计输入
场景标题：{{SCENE_TITLE_CN}}
叙事目的：{{STORY_BEAT_CN}}
情绪曲线：{{EMOTIONAL_ARC}}
时长：{{DURATION}}秒
对白：
{{DIALOGUES}}

角色形象参考：
{{CHARACTER_IMAGES}}

## 输出要求

1. **镜头数量**
   - 本场景{{DURATION}}秒，生成{{SHOT_COUNT}}个镜头
   - 公式：时长 / 4，向上取整，最少2个

2. **景别分配**
   - 第1个镜头：全景/远景（建立环境）
   - 中间镜头：中景（主体动作）
   - 最后1个镜头：特写（情绪高潮）

3. **Veo提示词规则**
   - 结构：Subject + Action + Environment + Lighting + Camera
   - 禁用词：cinematic, epic, 4K, masterpiece, unreal engine, stunning
   - 长度：80-120英文单词
   - 动作必须带副词（slowly, intensely, frustratedly）
   - 光影必须具体（golden morning light, blue rim light）

4. **质检要求**
   - 自动检查禁用词
   - 检查提示词长度
   - 标注检查结果

输出JSON格式。
```

## 五、API设计

### 5.1 第3步 API

```javascript
// 生成故事节拍
POST /api/story-beats/generate
Request: {
  episodeId: string,
  episodeContent: string,
  characters: Array<{name, identity, traits}>,
  totalDuration: number
}
Response: {
  scenes: Array<{
    sceneNumber, sceneTitleCn, sceneTitleEn,
    storyBeatCn, storyBeatEn, durationSeconds,
    emotionalArc, dialogues
  }>
}

// 保存审核后的故事节拍
POST /api/story-beats/save
Request: {
  episodeId: string,
  scenes: Array<Scene>,
  auditStatus: 'approved' | 'modified'
}
Response: { success: true }

// 获取已保存的故事节拍
GET /api/story-beats/:episodeId
Response: { scenes: [...], auditStatus, lastModified }
```

### 5.2 第4步 API（升级）

```javascript
// 获取第3步数据作为输入
GET /api/storyboard/input-data/:episodeId
Response: {
  storyBeats: { scenes: [...] },
  characterImages: { 角色名: URL }
}

// 生成分镜（单个场景）
POST /api/storyboard/generate-scene
Request: {
  episodeId: string,
  sceneNumber: number,
  sceneData: Scene, // 来自第3步
  characterImages: Object
}
Response: {
  shots: Array<{
    shotNumber, shotTypeCn, shotTypeEn,
    duration, veoPrompt, visualCn, visualEn,
    cameraMovement, qualityCheck: { passed, errors }
  }>
}

// 批量生成（并行）
POST /api/storyboard/generate-all
Request: { episodeId: string }
Response: { jobId: string }

// 查询生成进度
GET /api/storyboard/progress/:jobId
Response: {
  totalScenes: number,
  completedScenes: number,
  currentScene: number,
  status: 'generating' | 'completed' | 'error'
}

// 保存最终分镜
POST /api/storyboard/save-final
Request: {
  episodeId: string,
  scenes: Array<{
    sceneNumber,
    shots: Array<Shot>
  }>
}
```

## 七、关键优势

| 优势 | 说明 |
|------|------|
| **叙事质量** | 先定节拍再定镜头，确保故事结构完整 |
| **可控性** | 第3步审核叙事，第4步审核视觉，分层把关 |
| **灵活性** | 可调整场景顺序、时长，而不影响已生成分镜 |
| **复用性** | 第3步的故事节拍可用于其他形式改编（如漫画、广播剧） |
| **协作性** | 编剧专注第3步，导演专注第4步，分工明确 |
