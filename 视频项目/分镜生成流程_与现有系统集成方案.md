# 分镜生成流程优化方案
## 与现有短剧工作室系统集成

---

## 一、现有系统分析

### 1.1 当前6步工作流

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  1.选集  │ → │ 2.角色  │ → │ 3.分镜头 │ → │ 4.视频  │ → │ 5.配音  │ → │  6.导出  │
│         │    │  剧照   │    │         │    │  生成   │    │  制作   │    │         │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
```

**当前第3步"分镜头"的问题**：
- 一键生成，无中间控制
- 生成质量不稳定
- Veo提示词含禁用词
- 无人工审核环节

### 1.2 数据流转关系

| 步骤 | 输入 | 输出 | 下游依赖 |
|------|------|------|----------|
| 选集 | 小说章节 | 选中的集数列表 | 分镜头 |
| 角色剧照 | 角色列表 | 角色形象图 | 分镜头、视频生成 |
| **分镜头** | 集数内容+角色 | **分镜JSON** | **视频生成** |
| 视频生成 | 分镜+角色图 | 视频片段 | 导出 |
| 配音 | 分镜对白 | 音频文件 | 导出 |

**关键发现**：分镜头是核心枢纽，下游视频生成和配音都依赖它的输出质量。

---

## 二、集成方案设计

### 2.1 方案选择：升级现有"分镜头"步骤

**不是新增步骤，而是升级第3步**：

```
现有流程：
选集 → 角色剧照 → [分镜头：一键生成] → 视频生成 → 配音 → 导出

优化后流程：
选集 → 角色剧照 → [分镜头：三步生成+审核] → 视频生成 → 配音 → 导出
                              ↓
                    Step 3A: 生成故事节拍
                    Step 3B: 审核场景设计  ← 人工介入
                    Step 3C: 生成分镜细节
                    Step 3D: 审核分镜      ← 人工介入
```

### 2.2 新的"分镜头"步骤内部流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                      第3步：分镜头（升级后）                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  入口：已选集数 + 角色剧照数据                                        │
│                      ↓                                               │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │ Step 3A: AI生成故事节拍                                       │  │
│  │ • 输入：集数内容文本                                          │  │
│  │ • 调用：Gemini API 1次                                        │  │
│  │ • 输出：场景列表（无镜头细节）                                 │  │
│  │ • 展示：场景卡片（标题、情绪、对白预览）                        │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                      ↓                                               │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │ Step 3B: 人工审核场景设计 ⚠️ 关键控制点                        │  │
│  │ • 可编辑：场景标题、叙事目的、情绪曲线、对白                    │  │
│  │ • 可调整：场景顺序、时长分配                                   │  │
│  │ • 操作：[确认通过] / [返回修改]                                │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                      ↓                                               │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │ Step 3C: AI生成分镜细节                                       │  │
│  │ • 输入：审核后的场景设计                                      │  │
│  │ • 调用：Gemini API × N场景（并行）                             │  │
│  │ • 输出：完整分镜JSON（含Veo提示词）                            │  │
│  │ • 自动质检：禁用词检查、长度检查                               │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                      ↓                                               │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │ Step 3D: 人工审核分镜                                         │  │
│  │ • 可编辑：单个镜头的Veo提示词、视觉描述                        │  │
│  │ • 可重生成：单个场景重新调用AI                                 │  │
│  │ • 操作：[确认完成] → 进入第4步视频生成                         │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 四、技术实现

### 4.1 状态管理

```javascript
// 分镜头步骤的状态结构
const storyboardState = {
  // 当前步骤
  currentStep: '3C', // '3A' | '3B' | '3C' | '3D'
  
  // Step 3A 输出
  beats: [
    {
      sceneNumber: 1,
      sceneTitleCn: '法阵禁锢',
      sceneTitleEn: 'Array Imprisonment',
      storyBeatCn: '...',
      emotionalArc: '决绝→紧张',
      durationSeconds: 8,
      dialogues: [...]
    }
  ],
  
  // Step 3B 输出（审核后）
  sceneDesigns: [...], // 与beats结构相同，但可编辑
  
  // Step 3C 输出
  storyboardDraft: {
    scenes: [
      {
        sceneNumber: 1,
        shots: [
          {
            shotNumber: '1A',
            shotTypeCn: '全景',
            veoPrompt: '...',
            visualCn: '...',
            qualityCheck: { passed: true, errors: [] }
          }
        ]
      }
    ]
  },
  
  // Step 3D 输出（最终）
  storyboardFinal: {...}, // 审核后的最终版本
  
  // 生成状态
  generationProgress: {
    totalScenes: 10,
    completedScenes: 8,
    currentScene: 9,
    status: 'generating' // 'idle' | 'generating' | 'paused' | 'completed' | 'error'
  }
};
```

### 4.2 API设计

```javascript
// Step 3A: 生成故事节拍
POST /api/storyboard/generate-beats
Body: { episodeId, episodeContent, characters, totalDuration }
Response: { beats: [...] }

// Step 3B: 保存审核后的场景设计
POST /api/storyboard/save-scene-designs
Body: { episodeId, sceneDesigns }
Response: { success: true }

// Step 3C: 生成分镜细节（单个场景）
POST /api/storyboard/generate-scene-shots
Body: { episodeId, sceneDesign, characterImages }
Response: { shots: [...], qualityCheck: {...} }

// Step 3C: 批量生成（并行）
POST /api/storyboard/generate-all-shots
Body: { episodeId, sceneDesigns }
Response: { jobId } // 异步任务

// 查询生成进度
GET /api/storyboard/generation-progress?jobId=xxx
Response: { progress: 80, currentScene: 8, status: 'generating' }

// Step 3D: 保存最终分镜
POST /api/storyboard/save-final
Body: { episodeId, storyboardFinal }
Response: { success: true }

// 编辑单个镜头
POST /api/storyboard/update-shot
Body: { episodeId, sceneNumber, shotNumber, updates }
Response: { success: true }
```

### 4.3 Gemini Prompt优化

**Step 3A Prompt**（故事节拍）:
```markdown
基于以下集数内容，生成{{TOTAL_DURATION}}秒的故事节拍。

集数内容：
{{EPISODE_CONTENT}}

角色设定：
{{CHARACTERS}}

要求：
1. 按三幕结构：建立30%、对抗40%、高潮30%
2. 每个节拍包含：sceneTitleCn/En, storyBeatCn/En, emotionalArc, durationSeconds, dialogues
3. 对白要符合角色性格
4. 不要生成镜头细节

只输出JSON。
```

**Step 3C Prompt**（分镜细节）:
```markdown
基于以下场景设计，生成分镜。

场景设计：
{{SCENE_DESIGN}}

角色形象参考：
{{CHARACTER_IMAGES}}

要求：
1. 镜头数 = 场景时长 / 4，向上取整
2. Veo提示词规则：
   - 结构：Subject + Action + Environment + Lighting + Camera
   - 禁用词：cinematic, epic, 4K, masterpiece, unreal engine
   - 长度：80-120词
   - 动作必须带副词
3. 自动质检并标注问题

只输出JSON。
```

## 五、与下游步骤的衔接

### 5.1 第4步"视频生成"的输入

优化后的分镜JSON为视频生成提供：
- `veoPrompt`: 直接用于Veo生成
- `visualCn`: 用于理解画面内容
- `character`: 用于选择角色形象图作为首尾帧

### 5.2 第5步"配音制作"的输入

- `dialogues`: 对白文本
- `tone`: 语气指导
- `timestamp`: 时间轴对齐

## 六、实施建议

### 6.1 渐进式升级

| 阶段 | 内容 | 工作量 | 风险 |
|------|------|--------|------|
| Phase 1 | 只改Step 3C：优化Veo提示词生成+自动质检 | 2天 | 低 |
| Phase 2 | 增加Step 3A和3B：故事节拍+审核 | 3天 | 中 |
| Phase 3 | 增加Step 3D：分镜审核界面 | 2天 | 低 |

**建议**：先做Phase 1，验证效果后再做后续。

### 6.2 回滚方案

保留原有的"一键生成"按钮作为备选：
```
[🚀 智能分步生成]  [⚡ 快速一键生成（旧版）]
```

## 七、总结

| 维度 | 原方案 | 新方案 |
|------|--------|--------|
| 流程位置 | 独立5步页面 | 集成到第3步内部 |
| 人工介入 | 0次 | 2次（场景审核+分镜审核） |
| Gemini调用 | 1次 | 1次(节拍) + N次(分镜) |
| Veo提示词质量 | 不稳定 | 结构化+自动质检 |
| 用户体验 | 一键等结果 | 分步可控 |

**核心价值**：在现有工作流不变的前提下，通过升级"分镜头"步骤内部流程，提升生成质量。
